Руководство по XMLLevelWriter
XMLLevelWriter – приложение, которое предназначено для создания уровней для игры myRiseOfAtlantis. Эту программу я разработал на языке C++, в системе программирования C++ Builder. 
После запуска XMLLevelWriter с левой стороны окна приложения находится 121 ячейка, то есть максимальное их количество в игре. Первоначально они пустые, то есть, их нет. Для того чтобы создать ячейку и/или какой-либо объект, надо щёлкнуть на нужной ячейке правой клавишей мыши и выбрать в контекстном меню тип объекта. Опишу список возможных объектов, так как в игре объекты не имеют названий, а в конструкторе имеют только названия, без соответствующего изображения. 
Начну сверху:
1.	space – удаляет ячейку;
2.	cell – создает ячейку пола, без объекта находящегося на ней;
3.	ruby - создает ячейку пола с объектом - рубином на ней;
4.	helmet - создает ячейку пола с объектом – шлемом на ней;
5.	gold - создает ячейку пола с объектом – золотой монеткой на ней;
6.	green sun - создает ячейку пола с объектом – зеленым солнцем на ней;
7.	seashell - создает ячейку пола с объектом – ракушкой на ней;
8.	sun part - создает ячейку пола с объектом – частица солнца – имеет порядковый номер;
9.	lock – активна только при щелчке на уже созданном объекте, так как создаёт объект – замок, запирающий созданный объект;

Таким образом, ячейки, являющиеся пустыми (то есть отсутствующие в игре) в конструкторе отображаются серым цветом, в то время как активные ячейки, представляющие в игре какой-нибудь объект, отображаются белым цветом, и имеют надпись, повествующую о том, какой объект на них находится. Ячейки, представляющие в игре фишки под замком (закрытые фишки) в конструкторе отображаются чёрным с белой надписью, какой объект находится под замком.

Справа от ячеек находятся поля ввода. Они служат для задания изображения (текстуры) к определенному типу объекта. Когда создаются части солнца (мозаики), соответственно добавляются поля ввода для задания текстуры для нового кусочка. Однако в игре я жестко задал положения в мозаике для первых четырех кусочков (по порядковому номеру). Поэтому, в игре не может быть больше четырех кусочков. Собственно, в оригинале предел такой же. То есть, когда я изучал оригинальную The Rise Of Atlantis, я не на одном уровне не заметил больше четырех кусочков, везде 3 - 4.
При этом в конструкторе можно создать неограниченное количество кусочков. Но, в, то, же время, в окне умещается только десять дополнительных полей ввода для задания текстур для кусочков. Этого с лихвой хватит для любого уровня! Кроме того, имеется широчайший диапазон для совершенствования!

Естественно, уровень сохраняется в файле формата *.xml, имеющего определенную структуру. С помощью XMLLevelWriter можно открыть уже созданный уровень и/или сохранить под другим именем.
